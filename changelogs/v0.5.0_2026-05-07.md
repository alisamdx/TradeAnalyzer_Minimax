## Phase 5: Cache Management + Wheel Columns + Real-Time WebSocket

### Cache Management System (Priority 1: Foundation)

**Database Migration:**
- Created `004_cache_management.sql` with `cache_metadata` table (`last_screener_run`, `record_count`)

**Backend (Main Process):**
- Created `CacheManager` service (`src/main/services/cache-manager.ts`) to track cache staleness
- Implements staleness check on app startup (emits `cache-stale` if data > 1 hour old)
- Created IPC handlers: `cache:getStatus`, `cache:refresh`, `cache:updateLastRun`

**Frontend:**
- Built `CacheStatusIndicator` component (Green=Fresh, Red=Stale, with timestamp)
- Created `useCacheStatus` hook (polls every 5 minutes)
- Added cache indicator to Watchlist and Screener view headers
- Added "Refresh All Data" button in Watchlist view
- Implemented auto-refresh trigger when stale data detected

### Watchlist Wheel Columns (Priority 2: High Value)

**Backend:**
- Created `WheelCalculator` service (`src/main/services/wheel-calculator.ts`)
- Wheel Suitability Score formula: `ROEĂ—30% + D/E_QualityĂ—30% + MarketCapĂ—20% + StabilityĂ—25%`
  - ROE Component: min(ROE/30 Ă— 100, 100) Ă— 0.30
  - D/E Quality: if D/E < 1.0: 100, else: max(0, 100 - (D/E-1)Ă—50) Ă— 0.30
  - Market Cap: >$50B:100, >$10B:75, >$1B:50, else:25 Ă— 0.20
  - Stability: 25% placeholder for volatility component
- Target Strike calculation: `Current Price Ă— 0.92` (8% OTM)
- Est. Premium calculation: `Strike Ă— 1.2%` (30 DTE approximation)
- Updated `Quote` type and cache to include new fields

**Frontend:**
- Added columns to Watchlist table:
  - Wheel Suitability (0-100 score with color coding)
  - Target Strike
  - Est. Premium
- Created `useSortable` hook for sortable columns (all watchlist columns)
- Added column headers with sort indicators (asc/desc arrows)

### Real-Time Price Streaming (Priority 3: Critical)

**Backend:**
- Added `WebSocketService` (`src/main/services/websocket-service.ts`) in main process
- Connects to Polygon delayed WebSocket: `wss://delayed.polygon.io/stocks`
- Implements message handlers: `T` (Trade), `A` (Aggregate)
- Reconnection logic with exponential backoff: 3s, 6s, 12s, 24s, 48s
- Created IPC handlers: `websocket:subscribe`, `websocket:unsubscribe`
- Auto-subscribes to tickers when watchlist becomes active

**Frontend:**
- Built `RealtimePriceTicker` banner component for Analysis view
- Updated watchlist table with live price updates (green/red color coding)
- Created `useWebSocket` hook for connection management
- Added connection status indicator in status bar
