# v0.22.1 — 2026-06-01

## feat: Batch Jobs — global session toggle

### What changed

Added an in-memory global toggle to enable or disable all batch jobs for the current app session.

**Behavior:**
- Defaults to **ON** on every app launch (never persists)
- When OFF: startup jobs and scheduled (daily) jobs are skipped entirely
- When OFF: manual "Run Now" still works — only automatic triggers are blocked
- Flipping back ON immediately re-enables automatic triggers for the rest of the session

**Why:** During development, restarting the app repeatedly would trigger 30-second-delayed startup jobs on every launch. The toggle lets you disable automatic jobs at startup without permanently reconfiguring them.

### Backend changes
- `src/main/services/batch-service.ts`
  - Added `private batchJobsEnabled = true`
  - Added `setEnabled(enabled: boolean)` and `isEnabled(): boolean` public methods
  - `runStartupJobs()` returns early (with log) when toggle is OFF
  - `checkSchedule()` returns early when toggle is OFF

- `src/main/ipc/ipc-batch.ts`
  - Added `batch:set-enabled` handler — calls `batchService.setEnabled(bool)`
  - Added `batch:get-enabled` handler — returns `batchService.isEnabled()`

- `src/preload/index.ts`
  - Added `setEnabled(enabled: boolean)` and `getEnabled()` to `window.api.batch`

### UI changes
- `src/renderer/src/views/BatchView.tsx`
  - Header now has a toggle switch (green "Jobs ON" / red "Jobs OFF") aligned right of the title
  - Amber warning banner appears below the header when jobs are disabled: "Startup and scheduled jobs will not run until re-enabled. Manual 'Run Now' still works."
  - Toggle state initialized from `batch:get-enabled` on mount
