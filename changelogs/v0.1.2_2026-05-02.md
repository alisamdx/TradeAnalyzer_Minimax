**Summary**: Fix `rebuild:electron` so it actually swaps the better-sqlite3 binary's ABI. Discovered while verifying that `npm run dev` opens an Electron window (it didn't, until this fix).

## Bugs fixed

- **`rebuild:electron` was a no-op when a Node-built binary was already present.** v0.1.1 wired `rebuild:electron` to `electron-builder install-app-deps`, which delegates to `@electron/rebuild`. With `buildFromSource=false` (the default), `@electron/rebuild` reports "finished" but does not replace an existing `build/Release/better_sqlite3.node` whose ABI it considers "good enough". Result: after `npm run rebuild:electron`, the Node-target binary from a previous `pretest` was still in place, and the Electron app crashed on first DB call with `NODE_MODULE_VERSION 137 vs 128`.

## Changes

- **`scripts/rebuild-better-sqlite3.mjs`** (new) â€” small cross-platform helper that calls `prebuild-install --runtime=<electron|node> --force` from inside `node_modules/better-sqlite3`. For the Electron case, reads the actual installed Electron version from `node_modules/electron/package.json` (so an Electron version bump no longer requires a script edit).
- **`package.json`**:
