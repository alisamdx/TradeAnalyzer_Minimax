# v0.1.1 — 2026-05-02

**Summary**: Swap DB driver back from `node:sqlite` to `better-sqlite3` 12.x now that the dev environment has Visual Studio Build Tools 2022. No behavior change.

## Features added

None.

## Bugs fixed

None.

## Breaking changes

None — the swap is contained to `src/main/db/connection.ts`. Service-layer code only ever used the lowest-common-denominator API (`prepare`, `run`, `get`, `all`, `exec`, plus `withTransaction` from the wrapper), so nothing downstream needed to change.

## Changes

- **`package.json`**:
  - Added `better-sqlite3@^12.9.0` to `dependencies` and `@types/better-sqlite3@^7.6.13` to `devDependencies`.
  - Added `rebuild:electron` (= `electron-builder install-app-deps`) and `rebuild:node` (= `prebuild-install --runtime=node --force`) scripts.
  - Wired `predev`, `prebuild`, `prepackage` → `rebuild:electron` so launching the app always uses the Electron-ABI binary.
  - Wired `pretest` → `rebuild:node` so `npm test` always uses the system-Node-ABI binary.
  - Intentionally **no `postinstall`**: tests run cleanly out of a fresh `npm install` (prebuild-install puts the Node binary in place), and `dev`/`build` swap to the Electron binary on demand.
  - Bumped version `0.1.0` → `0.1.1`.
- **`src/main/db/connection.ts`**: rewritten to import `better-sqlite3` directly. Replaced the `createRequire('node:sqlite')` workaround. `openDatabase` now uses `db.pragma(...)` (idiomatic better-sqlite3) instead of `db.exec('PRAGMA ...')`. `withTransaction` is unchanged — kept as a thin wrapper so callers don't depend on `db.transaction(fn)()`.
- **`vitest.config.ts`**: removed `pool: 'forks'` and `server.deps.external: [/^node:/, 'sqlite']`. Both existed only to dodge Vite's `node:sqlite` resolution bug. Vitest defaults are fine again.
- **`README.md`** + **`AI_CONTEXT.md`**: updated Tech Stack, prerequisites, design-decisions, and Known Quirks sections to reflect the swap back. Added a row to `AI_CONTEXT.md → Recent Changes`.

## Verification

- `npm test` — all 37 tests still pass.
- `npm run typecheck` — both `tsconfig.node.json` and `tsconfig.web.json` pass.
- `npm install` re-runs `electron-rebuild` cleanly against Electron 32 / Node 20 ABI.

## AI session notes

- **Why two rebuild scripts.** The first attempt at this swap used a single `postinstall: electron-builder install-app-deps`. That left the binary built for Electron 32's Node 20 ABI (NODE_MODULE_VERSION 128), which broke vitest under system Node 24 (137). Splitting into `rebuild:node` + `rebuild:electron`, wired into `pretest` and `predev`/`prebuild`/`prepackage`, lets both `npm test` and `npm run dev` work from any starting state without the user having to think about it. Costs ~3-5s before each `dev` or `test` run for the binary swap; acceptable.
- **Why no `postinstall`.** Could have wired `postinstall: rebuild:node` to default to the test-friendly state, but it doubles install time and the `pretest` hook handles it anyway. After `npm install`, `prebuild-install` (better-sqlite3's own install hook) drops the Node binary in place, so tests already work.
- **Service-layer types didn't need to change.** The two `as unknown as Row[]` double-casts in `WatchlistService.list()` and `WatchlistService.listItems()` still typecheck because better-sqlite3's untyped `prepare(...)` returns `Statement<unknown[], unknown>`. If we ever want to drop those double-casts, switch to better-sqlite3's generic overload: `db.prepare<[], WatchlistRow>(sql).all()`. Deferred — not worth the diff right now and would re-tie callers to better-sqlite3-specific generics.
- **`tests/helpers.ts` needed an explicit return type.** TS4058: a function exporting a value whose type comes from a transitively-imported package can't infer the return type cleanly. Fixed by importing `DbHandle` from `connection.ts` and annotating `makeService(): { db: DbHandle; service: WatchlistService }`.
- VS Build Tools were installed by the user between v0.1.0 and v0.1.1; first verified by a successful `electron-builder install-app-deps` rebuild of `better-sqlite3` against Electron 32.3.3.
