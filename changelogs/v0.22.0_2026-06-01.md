# v0.22.0 — 2026-06-01

## feat: Analysis Engine — run all modes in one pass

### What changed

**Before:** Select watchlist → select mode → run → see results for one mode. Snapshots in sidebar.

**After:** Select watchlist → Run Analysis → all 5 modes run automatically → strategy tabs to switch results → snapshots listed below.

### UI changes
- `AnalysisView.tsx` — complete redesign: top controls row (watchlist dropdown + Run button), progress bar showing current mode + ticker, strategy tab strip (📈 Buy / 💰 Options Income / 🎯 Wheel / 🐂 Bullish / 🐻 Bearish) with result counts, results table below tabs, Past Runs section at bottom (newest first, per-row 🗑 delete, Clear All button)
- Mode selector cards removed
- Snapshots show mini badges per mode (e.g. 📈83 💰71 🎯45 🐂31 🐻19) and ticker count
- Clicking a past run loads all 5 modes into the tabs

### Backend changes
- `src/main/services/analysis-service.ts`
  - Added `runMode()` private helper (inner loop, no cancel reset)
  - `analyzeWatchlist()` refactored to use `runMode()`
  - Added `analyzeWatchlistAllModes(tickers, onProgress)` — runs all 5 modes sequentially, respects cancel between modes
  - Added `saveAllModesSnapshot(watchlistId, tickerCount, results)` — stores `mode='all'` snapshot with grouped payload
  - Added `getAllModesSnapshot(id)` — decodes 'all' snapshot payload
  - Mode casts updated to `AnalysisMode | 'all'`

- `src/main/ipc/ipc-analysis.ts`
  - Added `analysis:run-all` handler
  - Added `analysis:get-all-modes-snapshot` handler

- `src/preload/index.ts`
  - Added `runAll(watchlistId, tickerSubset?)` and `getAllModesSnapshot(id)` to `window.api.analysis`
  - `onProgress` callback typed with `AnalysisProgressEvent` (includes optional `mode` field)

### Type changes (`src/shared/types.ts`)
- `AnalysisSnapshotRow.mode` widened to `AnalysisMode | 'all'`
- Added `AnalysisAllModesPayload` interface
- Added `AnalysisAllModesRunResult` interface
- Added `AnalysisProgressEvent` interface (replaces inline type in preload)
