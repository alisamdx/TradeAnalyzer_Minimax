# v0.15.0 — 2026-05-26

## Dual-Provider Architecture (Polygon + E*Trade)

### New: `OptionsProvider` Interface
- `src/main/services/options-provider.ts` — new interface with four methods:
  - `getOptionsExpirations(ticker)` — real expiry list (E*Trade) or `[]` (Polygon → falls back to generated Fridays)
  - `getOptionsChain(ticker, expiration)` — full calls + puts chain
  - `getOptionsIV(ticker)` — ATM IV + 52-week high/low as percentages
  - `getOptionsIVAndPremium(ticker, targetExpiry, targetStrike)` — combined IV + nearest put premium

### New: `ETradeDataProvider`
- `src/main/services/etrade-data-provider.ts` — implements `OptionsProvider` using E*Trade Market API
- Reads live OAuth tokens from `secure-settings.ts` via factory callback (always uses latest, handles midnight expiry)
- E*Trade IV is a decimal fraction (0.3882); multiplied by ×100 to match the Polygon convention (38.82%)
- `getOptionsExpirations` returns real E*Trade expiry dates sorted ascending
- `getOptionsIVAndPremium` resolves ATM IV from nearest expiration, approximates 52-week range from snapshot spread

### Updated: `PolygonDataProvider`
- Now implements both `DataProvider` and `OptionsProvider`
- Added `getOptionsExpirations` (returns `[]`; callers generate next N Fridays as fallback)
- `readonly name = 'polygon'` for provider identification

### Wiring: `index.ts`
- Reads `optionsProvider` setting from DB at startup (`'polygon'` default)
- If `'etrade'`: instantiates `ETradeDataProvider` with credentials from `secure-settings`
- If `'polygon'`: uses existing `PolygonDataProvider` directly
- Injects chosen `OptionsProvider` into `ipc-options.ts`, `ipc-leaps-csp.ts`, `AnalysisService`

### Updated: `LeapsCspService`
- Constructor now takes `DataProvider` (for `getQuote`, `getFundamentals`, `getHistoricalBars`) and `OptionsProvider` (for `getOptionsChain`, `getOptionsIVAndPremium`) as separate args
- Market gate SPY/VIX/HYG/IEF quotes always use `DataProvider` (Polygon)
- LEAPS and CSP contract selection uses `OptionsProvider` (switchable)

### Updated: `AnalysisService`
- Accepts optional `OptionsProvider` as 5th constructor arg
- Uses it (falling back to `DataProvider`) for `getOptionsIV` and `getOptionsChain` calls in analysis modes

### Settings UI
- "Options Data Source" dropdown in Settings → API & Data tab
- Choices: Polygon.io (default) / E*Trade
- Saved immediately to DB; **restart required** to apply
- `window.api.settings.getOptionsProvider()` / `setOptionsProvider()` preload bridge
- `settings:get-options-provider` / `settings:set-options-provider` IPC handlers
