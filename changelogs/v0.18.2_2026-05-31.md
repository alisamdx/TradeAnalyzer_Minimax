# v0.18.2 — 2026-05-31

## ENH-3: Portfolio Greeks Monitor

### What changed

Added an aggregate Greeks summary bar to `PortfolioView`, shown above the Open/Closed/Advisor tabs whenever there are open positions.

### Summary bar metrics

| Chip | Formula | Source |
|------|---------|--------|
| **Net Δ** | Σ (et.delta × multiplier) across all open positions | E*Trade sync |
| **Θ/day** | Σ (et.theta × multiplier) for options only; in $ | E*Trade sync |
| **Vega** | Σ (et.vega × multiplier) for options only | E*Trade sync |
| **BP Used %** | totalCapitalDeployed / accountSize × 100 | Settings + P&L summary |
| **Exp buckets** | Count of positions expiring ≤7d / ≤14d / ≤21d | position.expirationDate |

Multiplier = `quantity × 100` for CSP/CC, `quantity` for Stock. BP% is hidden until `accountSize` is configured.

### Files changed

- `src/renderer/src/views/PortfolioView.tsx` — `GreeksChip` component, `greeksSummary` useMemo, `accountSize` state, settings fetch in `loadData`, Greeks bar render
- `src/main/ipc/ipc-settings.ts` — `accountSize: number` added to `AppSettings`, `DEFAULT_SETTINGS`, and `get-all` parse handler
- `src/shared/types.ts` — `accountSize: number` added to `AppSettings`
- `docs/formulas.md` — `#portfolio-greeks` anchor added

### No new API calls or migrations

Greeks come from the existing E*Trade sync (`etrade_positions` table). `accountSize` is a plain `settings` key/value row — no migration required.
