# v0.4.0 ‚Äî 2026-05-02

**Phase 4: Validation Dashboard Polish + Settings/Diagnostics**

## Features Added

- **Chart pattern callouts** ‚Äî Pattern labels (doji, hammer, engulfing, morning/evening star) rendered as HTML markers at the correct price/time position using `series.priceToCoordinate()`. Re-renders on timeframe change and zoom.
- **Entry zone band** ‚Äî Entry zone high/low rendered as horizontal line overlays (green) on the chart.
- **Target line** ‚Äî Suggested target price rendered as a dashed amber line on the chart.
- **Stop loss line** ‚Äî Stop loss rendered as dashed red line on the chart (already existed, now using consistent helper).
- **Supply/demand zone labels** ‚Äî Zone prices labeled on the right edge of the chart with `DEMAND`/`SUPPLY` + price.
- **EPS sparkline** ‚Äî Small SVG sparkline in Section A (Executive Summary) showing the last 4 quarters of EPS estimates vs actuals.
- **Settings view** (`SettingsView.tsx`) ‚Äî Full Settings panel with 5 sections:
  - **General** ‚Äî Rate limit presets (Conservative 50 / Default 100 / Aggressive 250 rpm), log retention days, error log retention days, auto-backup toggle.
  - **API & Data** ‚Äî Polygon API key (masked, show/hide toggle), open logs directory button.
  - **Cache & Rate Limits** ‚Äî Per-cache TTL inputs (quote, fundamentals, options chain) with human-readable display.
  - **Diagnostics** ‚Äî Run self-check button, subsystem status table (schema version, DB file size, quote/fundamentals cache hit rates, log directory size, recent errors, app version), overall OK/degraded badge.
  - **Backup & Restore** ‚Äî Backup Everything (copies DB + logs + AI_CONTEXT to a timestamped backup folder), Restore from Backup (file picker for `.sqlite` / `.db` files).
- **Settings IPC** (`ipc-settings.ts`) ‚Äî `settings:get`, `settings:set`, `settings:get-all`, `settings:set-all`, `settings:get-api-key`, `settings:set-api-key`, `settings:open-logs-dir`, `settings:backup-everything`, `settings:restore-backup`.
- **Diagnostics IPC** ‚Äî `diagnostics:run` ‚Äî pings all subsystems, returns per-check status + overall verdict.

## Bugs Fixed

- `detectEveningStar` guard was inverted ‚Äî checked `min(o,c) < mid` (should be `> mid` for upper half). Also fixed the mirror test case to use `max(o,c)` with values above the midpoint. `detectMorningStar` was already correct.
- `package.json` dev script guards against `ELECTRON_RUN_AS_NODE=1` in the shell environment (causes `cli.js` to incorrectly spawn `node` which runs the Electron app bundle as ESM ‚Üí `TypeError: Cannot read properties of undefined (reading 'exports')`). Fixed with `env -u ELECTRON_RUN_AS_NODE electron-vite dev`.

## Breaking Changes

- None ‚Äî Phase 4 adds new routes and views only.

## AI Session Notes

- `priceToCoordinate()` is on `ISeriesApi`, not `IChartApi`. Always use the candle series to convert prices to coordinates.
- `subscribeVisibleLogicalRangeChange` returns `void` (not an unsubscribe fn). Use the handler ref and call `unsubscribeVisibleLogicalRangeChange(handler)` on cleanup.
- `BrowserWindow.getFocusedWindow() ?? BrowserWindow.getAllWindows()[0]` is the pattern for dialog windows that may have no focused window.
- `TIMEFRAMES[2]` is the 6M default index ‚Äî `useRef<TimeframeOption | null>` avoids `undefined` at init.
- Pattern callouts: filter patterns by `bar.t >= cutoff` so they only appear within the visible timeframe window.
