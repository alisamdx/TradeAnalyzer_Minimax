# v0.17.0 — 2026-05-30

## IV History — True IV Rank & IV Percentile

### Core service (`src/main/services/iv-history-service.ts`)
- `computeAtmIv()`: 30-day constant-maturity ATM IV via DTE-weighted interpolation between the two expirations bracketing 30 DTE (average of call + put IV at the ATM strike nearest to underlying price)
- `IvHistoryService.storeReading()`: upserts one (ticker, date) reading into `iv_history`
- `getIvRank(ticker)`: returns IV rank + IV percentile over last 252 trading days (requires ≥ 21 data points)
- `getCoverage(universe)`: complete/partial/none counts + last refresh date + total readings
- `getGaps(universe)`: generates missing (ticker, date) pairs from each ticker's last reading to yesterday ET
- `runBackfill(phase, onProgress)`: orchestrates MarketData.app fetches for `initial_sp500`, `initial_russell`, or `gap_fill` phases; streams progress events
- `captureFromEtradeChain()`: silently stores today's ATM IV from any E*Trade options chain fetch — no extra API call
- `getInitialLoadStatus()`: badge state for Step 1/2 in the management screen
- NYSE holiday calendar hardcoded for 2023–2027; `tradingDaysInRange()` / `tradingDaysAgo()` helpers

### MarketData.app provider (`src/main/services/marketdata-provider.ts`)
- `MarketDataProvider`: HTTP client for MarketData.app historical options chains
- Dedicated `TokenBucketRateLimiter` at 50 RPM (separate from Polygon)
- Parses parallel-array JSON response; handles `no_data`, 429 rate-limit, and error responses
- Token retrieved at call time from encrypted secure storage (`marketdataApiToken`)

### Database migration (`migrations/016_iv_history.sql`)
- `iv_history` table: `(ticker, date)` unique, `atm_iv`, `underlying_px`, `exp_near`, `exp_far`, `dte_near`, `dte_far`, `source`
- Indexed on `ticker` and `date`

### IPC layer (`src/main/ipc/ipc-iv-history.ts`)
- `iv-history:get-coverage` — coverage summary for a universe
- `iv-history:get-gaps` — gap summary (pair count, date range)
- `iv-history:start-backfill` — launches backfill; streams progress via `iv-history:progress`
- `iv-history:cancel` — cancels running backfill
- `iv-history:get-rank` / `iv-history:get-ranks` — single and batch IV rank queries
- `iv-history:get-initial-load-status` — badge state for management screen
- `iv-history:save-token` / `iv-history:get-token-configured` — MarketData.app token storage
- `iv-history:set-rate` — update MarketData.app rate limit at runtime

### Preload bridge (`src/preload/index.ts`)
