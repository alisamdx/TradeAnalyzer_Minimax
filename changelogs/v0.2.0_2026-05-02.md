# v0.2.0 — 2026-05-02

**Phase 2: Index Screener** — S&P 500 / Russell 1000 screener with fundamentals-first filtering.

## Features Added

- **FR-2 Screener** — Full index screening engine: S&P 500, Russell 1000, or both (deduplicated union). Universe sizes shown before run. Last-refresh timestamp per constituent list.
- **17 default filters** matching spec §5.2.2 table (market cap, P/E, EPS, revenue growth, EPS growth, D/E, ROE, profit margin, FCF, current ratio, avg volume, avg option vol, price, 52-wk distance, beta, earnings exclude, sector exclude).
- **Strict + Soft-match modes** — Strict: must pass all enabled filters. Soft: rank by pass score, show all candidates.
- **Screen presets (FR-2.5)** — Save/load/delete named filter presets. "Default" ships out of the box.
- **Run history** — Past runs are stored and can be re-opened without re-fetching.
- **Save as watchlist (FR-2.4)** — Multi-select screen results and save as a new watchlist in one click.
- **Wikipedia constituent refresh** — Respectful single-fetch from Wikipedia S&P 500 / Russell 1000 pages. Cached 7 days in SQLite.
- **Fundamentals computer** — Derives P/E, EPS, ROE, D/E, profit margin, revenue growth, EPS growth, FCF, current ratio from raw Polygon `/vX/reference/financials` data. All formulas documented in `docs/formulas.md`.
- **Quote cache** — 60s TTL, `QuoteCache` class in `src/main/services/cache-service.ts`. Auto-refresh on watchlist view (60s interval). Manual ↻ button.
- **Fundamentals cache** — 24h TTL, `FundamentalsCache` class. Used by screener to avoid redundant Polygon calls.
- **API + error logging (EP-3, EP-4)** — JSON lines to `logs/api/` and `logs/errors/`, daily rotation, 30/90-day retention. API keys scrubbed. `pruneOldLogsOnStartup()` runs on each launch.

## AI Session Notes

- `node:fetch` is **not** a valid module specifier in Node 24 (even though `fetch` itself is a global). Uses the implicit global `fetch` in the Electron main process.
- `QuoteSnapshot` (data-provider interface) and `Quote` (shared types) both now include `fetchedAt: string` — needed for cache TTL checking.
- Migration test updated to be migration-count-agnostic (Phase 2 adds migration 002).
- `TickerScreenData.passScore` added to match `ScreenResultPayload.passScore` in shared types.
- `PolygonDataProvider.getIndexConstituents` is a no-op — constituents are loaded from bundled CSVs by `ConstituentsService`, not through the Polygon provider.
- `ScreenerService` is instantiated with a `getConstituents` closure in `main/index.ts` to avoid circular import issues between services.
- `initCacheTables()` is called separately from migrations to avoid re-declaring tables that exist in both the migration SQL and the cache service — the `CREATE TABLE IF NOT EXISTS` pattern is safe.
