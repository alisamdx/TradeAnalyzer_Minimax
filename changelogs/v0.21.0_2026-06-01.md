# v0.21.0 — 2026-06-01

## Automated Batch Jobs

Three new daily background jobs registered at startup alongside the existing Daily IV Capture:

| Job ID | Name | Trigger |
|--------|------|---------|
| `daily-market-sync-stocks` | Daily Market Sync (Stocks) | Startup +60s · 16:30 ET |
| `daily-market-sync-etfs` | Daily Market Sync (ETFs) | Startup +90s · 16:30 ET |
| `daily-price-gap-fill` | Daily Price Gap Fill | Startup +120s · 16:30 ET |

All three are registered via the extended `BatchService.registerJob()` which now accepts `{ dailyScheduleTime, runOnStartup, startupDelaySeconds }`. They chain sequentially in a single scheduler tick because `checkSchedule()` awaits each `runJob()`.

### MarketSyncJob (`src/main/services/jobs/market-sync-job.ts`)
- Skips non-trading days via `isTradingDay()`
- Delegates to `screenerService.syncUniverse(universe)` — same path as the manual Data Sync button
- `universe` is `'both'` (stocks) or `'etf'`

### PriceGapFillJob (`src/main/services/jobs/price-gap-fill-job.ts`)
- SQL: `SELECT ticker FROM historical_prices GROUP BY ticker HAVING MAX(date) < date('now', '-2 day')`
- Calls `fetchAndStorePrices(service, provider, ticker, '1M')` with 500ms sleep between tickers
- No trading-day guard — catches up on missed days any time the app runs

## E*Trade Token Auto-Renewal

`DailyIvCaptureJob` now handles two token failure modes:
- **Dormant (inactive > 2h, same day)**: calls `optionsProvider.renewToken()` transparently, retries connectivity check
- **Expired (midnight rollover)**: returns a `notification` with `type: 'error'`, message, and `cta: { label: 'Reconnect', view: 'settings' }`

`ETradeDataProvider.renewToken()` added — delegates to `renewAccessToken(getCreds())`.

`BatchJobResult` extended with `notification?: AppNotification` so any job can surface a custom CTA without relying on the generic success/fail notification.

## BatchView — Live Activity Panel

A new panel appears between the Jobs table and Run History while a job is running:

- **Header**: spinning ⟳ Live Activity · live counter (`142 / 1247 tickers · ✓ 89 — 41 ✗ 12`)
- **Progress bar**: animated blue fill → turns green on completion
- **Ticker log**: monospace, one line per event (`✓ AAPL updated`, `— RIVN skipped — No expirations`, `✗ SPCE failed — 401`)
- Max 300 events buffered; auto-scrolls to bottom; auto-clears 3s after job finishes
- Panel is invisible when no job has run in the session

## Infrastructure

- `app.setLoginItemSettings({ openAtLogin: true })` — app auto-launches on Windows login
- Production DB path: `app.isPackaged ? app.getPath('userData') : join(appPath, 'data')` — fixes packaged exe finding no DB
- `npm run package` now sets `CSC_IDENTITY_AUTO_DISCOVERY=false` and `build.win.forceCodeSigning: false` to skip code signing and avoid winCodeSign symlink errors on Windows. **Requires Windows Developer Mode** (Settings → System → For developers → on) — one-time setup.
