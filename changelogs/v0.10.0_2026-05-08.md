# v0.10.0 — 2026-05-08

## Settings Enhancements (Priority 9)

### Overview
Extended the settings system with user preferences for alerts, connectivity theming, and keyboard navigation. All settings are persisted to the database and apply immediately or on next app launch.

### New Settings

**soundAlertsEnabled** (boolean, default: `true`)
- Controls whether sound notifications play when alerts are triggered
- UI: Toggle in Settings → General
- Usage: When enabled, the app will play a sound for price alerts, expiration warnings, and delta breaches

**autoConnectWebSocket** (boolean, default: `true`)
- Controls whether the WebSocket connects automatically on app startup
- UI: Toggle in Settings → General
- Technical: Main process checks this setting before calling `wsService.connect()`
- When disabled, user must manually connect via UI controls (to be implemented)

**defaultScreenerIndex** (enum: 'sp500' | 'russell1000' | 'both', default: `'sp500'`)
- Determines the default universe when opening the Screener view
- UI: Dropdown in Settings → General
- Technical: ScreenerView loads this setting on mount and initializes universe/criteria accordingly

**theme** (enum: 'dark' | 'light', default: `'dark'`)
- Controls the application color scheme
- UI: Two-button toggle in Settings → General (Dark / Light)
- Technical: 
  - CSS variables defined in `:root` (dark) and `[data-theme="light"]` selector
  - App.tsx applies `data-theme` attribute to `document.documentElement`
  - All components use CSS variables (`var(--bg)`, `var(--text)`, etc.)

**keyboardShortcuts** (object)
- Configurable keyboard shortcuts for common actions
- UI: Text inputs in Settings → Keyboard tab (5 shortcuts)
- Default shortcuts:
  - Refresh Quotes: `F5`
  - Run Analysis: `Ctrl+Shift+A`
  - Open Screener: `Ctrl+Shift+S`
  - Open Portfolio: `Ctrl+Shift+P`
  - Open Briefing: `Ctrl+Shift+B`
- Storage: JSON-serialized object in settings table
- Note: Requires app restart to take effect (not yet implemented in this release)

### Technical Changes

**shared/types.ts**
- Extended `AppSettings` interface with 5 new fields
- Added new `KeyboardShortcutsConfig` interface

**main/ipc/ipc-settings.ts**
- Updated `DEFAULT_SETTINGS` with Priority 9 defaults
- Extended `get-all` handler to parse new settings (booleans, enums, JSON)
- Settings with special handling:
  - `keyboardShortcuts`: JSON.parse with fallback to defaults
  - `theme`/`defaultScreenerIndex`: Cast to typed enums
  - Booleans: Compare to `'true'` string

**main/index.ts**
- Conditional WebSocket connection:
```typescript
const autoConnectSetting = db.prepare("SELECT value FROM settings WHERE key = 'autoConnectWebSocket'").get();
const shouldAutoConnect = autoConnectSetting?.value !== 'false';
if (shouldAutoConnect) {
  wsService.connect();
}
```

**renderer/App.tsx**
- Added useEffect to apply theme on mount:
```typescript
useEffect(() => {
  window.api.settings.getAll().then((settings) => {
    if (settings.theme) {
      document.documentElement.setAttribute('data-theme', settings.theme);
    }
  });
}, []);
```

**renderer/views/ScreenerView.tsx**
- On mount, loads settings and sets default universe:
```typescript
window.api.settings.getAll()
  .then((settings) => {
    const defaultUniverse = settings.defaultScreenerIndex || 'sp500';
    setUniverse(defaultUniverse);
    setCriteria(defaultCriteria(defaultUniverse));
  });
```

### Backwards Compatibility

All new settings default to sensible values that maintain existing behavior:
- Sound alerts: enabled (new feature, no breaking change)
- WebSocket auto-connect: enabled (existing behavior preserved)
- Default screener: sp500 (existing default preserved)
- Theme: dark (existing color scheme preserved)
- Keyboard shortcuts: standard defaults

Existing installs will automatically use defaults on first access.
