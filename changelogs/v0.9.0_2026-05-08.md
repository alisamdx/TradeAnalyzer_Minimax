# v0.9.0 — 2026-05-08

## Morning Briefing Dashboard (Priority 7)

### Database
- Migration `007_alerts.sql` — alerts table for position and price alerts (reused for Priority 8)

### Backend: BriefingService
**Location:** `src/main/services/briefing-service.ts`

Market regime detection:
- Fetches SPY daily bars, calculates 20-day and 50-day SMA
- Trend classification: Bullish (Price > 20MA > 50MA), Bearish (Price < 20MA < 50MA), Neutral (mixed)
- Fetches VIX value, classifies: Low (<15), Normal (15-25), High (>25)
- Generates regime summary text for trader guidance

Action items generation:
- Queries positions expiring within 5 days (expiration warning)
- Queries positions with delta > |0.40| (risk management alert)
- Earnings alerts stubbed (Polygon has no public earnings endpoint)

Top setups query:
- Criteria: ROE > 15%, D/E < 1.0, Market Cap > $10B, FCF Yield > 0%
- Returns top 15 quality stocks with wheel metrics (suitability score, target strike, estimated premium)
- Results sortable by any column

### Frontend: BriefingView
**Location:** `src/renderer/src/views/BriefingView.tsx`

Components:
- **MarketRegimeCard**: Dual-badge layout showing SPY trend (color-coded: green bullish/red bearish/gray neutral) and VIX level (green low/red high/orange normal). Shows current price, SMA values, and regime summary text.
- **ActionItemsList**: Priority-sorted items with icons (🔴 high, 🟡 medium, 🔵 low). Shows ticker, alert type, details, and metadata (days remaining for expirations).
- **TopSetupsTable**: 15-row table with sortable columns (ticker, ROE, P/E, D/E, wheel score, target strike, est. premium). Wheel scores color-coded (green ≥70, orange ≥50, red <50).

Features:
- Auto-refresh on component mount and every 5 minutes
- Manual refresh button with loading state
- Error toast with dismiss
- Empty states for no action items / no setups

### IPC Handlers
**Location:** `src/main/ipc/ipc-briefing.ts`

- `briefing:getFull` — Returns complete briefing (market regime, action items, top setups)
- `briefing:refresh` — Force refresh of all briefing data

## Alerts System (Priority 8)

### Database
- Table `alerts` with columns: id, type, ticker, threshold, direction, triggered, triggered_at, created_at
- Types: 'price' (above/below target), 'position' (expiration, delta)
- Indexes on ticker, triggered status for efficient querying

### Backend: AlertsService
**Location:** `src/main/services/alerts-service.ts`

Features:
- CRUD operations: createAlert, listAlerts, deleteAlert, markTriggered
- Price alert checking against WebSocket price updates
- Supports direction: 'above' or 'below' for price thresholds

Integration:
- Price alerts checked in real-time via WebSocket price updates
- No polling needed — leverages existing WebSocket infrastructure

### Frontend
Alerts UI is embedded in Portfolio view (position-level alerts) and Briefing view (action items from alerts).

Toast notifications deferred to Priority 9 (Settings Enhancements) where sound alerts will be implemented.

### IPC Handlers
**Location:** `src/main/ipc/ipc-alerts.ts`

- `alerts:create` — Create new alert (price or position)
- `alerts:list` — List all active (non-triggered) alerts
- `alerts:delete` — Remove alert by ID
- `alerts:markTriggered` — Mark alert as triggered (called when threshold breached)
