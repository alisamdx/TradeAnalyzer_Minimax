# v0.3.0 — 2026-05-02

**Phase 3: Analysis Engine + Producer/Consumer Pipeline**

## Features Added

- **FR-3 Analysis Engine** — 5 analysis modes over any watchlist:
  - **Buy Opportunities** — Composite score 0–10 (SMA stack + RSI 40–65 + fundamentals). Entry zone, stop-loss, target, R:R ratio.
  - **Options Income** — CSP/CC at 30–45 DTE, delta 0.20–0.35. Annualized return, breakeven, capital required.
  - **Wheel Strategy** — IV rank ≥ 30, no near-term earnings, OI ≥ 500. Suitability score 1–10.
  - **Bullish Strategies** — ADX confirmation, recommends long call / bull call spread / short put.
  - **Bearish Strategies** — ADX confirmation, recommends long put / bear put spread / short call.
- **Producer/Consumer Pipeline (§4.4)** — `TokenBucketRateLimiter` (10–500 rpm, configurable), `JobQueue` (SQLite-backed, resumable), `ValidateAllService`.
- **FR-4.4 Validate All** — Deep-dive batch validation of every ticker in a watchlist: verdict (Strong/Acceptable/Caution/Avoid), fundamentals, trend, RSI, MACD, Bollinger, volume anomaly, IV data.
- **Snapshot history** — Analysis runs saved to `analysis_snapshots` table. Past snapshots viewable, results re-openable, saveable as watchlist.
- **New DB migration (003)** — `analysis_snapshots` (auto PK), `job_runs`, `job_progress` tables.
- **Full indicator library** — `computeSMA`, `computeEMA`, `computeRSI` (Wilder), `computeATR` (Wilder), `computeADX` (Wilder), Bollinger Bands, MACD, swing high/low. All formulas documented in `docs/formulas.md`.
- **Preload bridge** — `window.api.analysis` (listModes, run, getSnapshots, getSnapshot, saveAsWatchlist, cancel) + `window.api.validateAll` (run, getStatus, cancel) + `window.api.jobs` (listIncomplete, resume, discard).
- **AnalysisView** — React component: watchlist dropdown, 5 mode cards, run/cancel, progress bar, mode-specific results table, save-as-watchlist, export CSV, snapshot history panel.
- **Analysis nav button** — Added to sidebar alongside Watchlists and Screener.

## Bugs Fixed

- `ratePerMs` was `readonly` in `TokenBucketRateLimiter` preventing `setRate()` from updating it (TypeScript error).
- `smooth()` function return type was `number[]` but needed to be `(number | null)[]` — null padding at start of series was rejected by TypeScript.
- `smoothDX[idx]` guard added to avoid `undefined` to `null` assignment in ADX computation.
- `watchlistService.getItems` → `listItems` (API rename).
- Duplicate `declare global { interface Window { api } }` removed from `ScreenerView.tsx` and `AnalysisView.tsx` — consolidated in `global.d.ts`.
- `withTransaction` import removed from `validate-all-service.ts` (unused).

## Breaking Changes

- None — Phase 3 adds new routes only.
